Working Directory

  • The (current) working directory is the directory R is currently working in.

  • The working directory is the default location R uses when reading or writing a file.

  • When you open the econ1003.Proj file, it sets the working directory to be the one where the project file is located (i.e., econ1003 folder.)

  • Open the econ1003.Proj. The top of Console pane shows the working directory.

  • Or you can type in Console

getwd()

  • Confirm that the directory is the one where the project file is located.

  • To see contents in the current working folder in Files pane, click More file commands \(\rightarrow\) Go to Working Directory in the panel.

The working directory is the default location R uses when reading or writing a file.

library(tidyverse)
sale <- read_rds("seoul_apt_sale.rds")

The command above does not work, since R cannot find the file in the current working folder.


Instead, use

library(tidyverse)
sale <- read_rds("data/seoul_apt_sale.rds")

Confirm in Files pane that the seoul_apt_sale.rds is in data subfolder from the working directory.

Note we use ‘/’ rather than ‘\’.

Relative Path

This is called a relative path.

  • A relative path is shorter than an absolute path.

  • It is more portable. For example, if you move the whole working directory to another location, all relative paths within the folder would still work.

  • Easier to share codes with others (or if you have multiple computers). Different computers have different directory structures.

Example

ggplot(mtcars, aes(mpg, hp)) +
    geom_point()

The following saves the plot in econ1003, the current working folder.

ggsave("mpg_hp.png")


If you want to save the plot in temp subfolder, use

ggsave("temp/mpg_hp.png")

Confirm in Files pane, that the files are created in the two folders.
